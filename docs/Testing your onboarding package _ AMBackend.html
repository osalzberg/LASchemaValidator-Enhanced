Testing your onboarding package

Code review

Please start by checking in your team's onboarding folder via a standard PR procedure directly into the master branch. One of the Log Analytics team members will assist you in reviewing your schema and transforms. When one of the team members signs off, the PR will automatically be completed.

Static validation

Once your PR is approved, a build will be kicked off, and these unit tests automatically executed. The build pipeline can be monitored here. Any issues with unit tests as well as other static tests will be surfaced up through the build. You can generally ignore warnings.

Sample input validation

As part of the build being generated, a step exists that runs your transforms against the sample input files you provided. This is used for the static validation step above to make sure that names, data types, and similar are consistent. As an extra step, however, you can see the raw output of your transformation being applied to ensure that it matches what you expect. To do so, start by opening the build artifact associated with your PR. There, click on "[nn] published" under the "related" header:

Navigate to /drop/test/_post_command__execute_kql_transform_tests/_testresults/bin/KqlTransform/NGSchemas/[RTFolderName]/TransformedRecords/[TableName_SampleInputFileName].json

If you see any issues here, please troubleshoot your transforms to resolve the issue. Don't hesitate to reach out to the onboarding team if you need any assistance with this.
